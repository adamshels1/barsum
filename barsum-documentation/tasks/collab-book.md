# Совместная книга (Collaborative Book / Цепочка глав)

## Суть

Новый тип книги, которую **дети (и родители) сочиняют вместе под кураторством эксперта**, глава за главой, голосом.

> **Ключевое архитектурное решение:** «автор продолжения» с самого начала **полиморфный** (`authorType: 'child' | 'parent'` + `authorId`), ровно как устроен монетный реестр (`fromType/toType`). Поэтому участие родителя ничего не ломает — детский флоу это частный случай `authorType='child'`. UI для родителя можно выкатить позже, модель уже готова.

Коротко цикл:
1. **Эксперт** создаёт книгу-заготовку: название, обложка, **первая глава** (текст + опц. аудио-озвучка). Проходит модерацию администратора → публикуется в раздел «Сочиняем вместе».
2. **Ребёнок** открывает книгу, читает/слушает текущую последнюю главу, нажимает **«Придумать продолжение»** и записывает аудио (до 60 сек) — как он видит продолжение.
3. Продолжений много — участвовать может **любой ребёнок**. Каждое продолжение транскрибируется и получает **ИИ-скор** (подсказка эксперту, не авто-решение).
4. **Эксперт** смотрит все продолжения раунда, **выбирает одно или несколько лучших**, может отредактировать текст. Выбранное становится **следующей главой**.
5. **Победители** получают монеты + пометку «соавтор». **Проигравшие** получают монеты за участие + **мотивирующий текст** (см. §7), чтобы вернулись в следующий раунд.
6. Цикл повторяется для новых глав, пока эксперт **не завершит книгу**.
7. По завершении — обложка (загрузка эксперта или ИИ-генерация) → книга публикуется в магазин как обычная читаемая книга; в описании перечислены **все соавторы** (дети и родители).
8. (Будущее, вне этой задачи) Популярные книги (лайки/прочтения) → мультфильм / аниме / сериал.

**Почему это ценно:** переиспользует ~90% существующей инфраструктуры (запись аудио, транскрибация `ai.service`, очередь эксперта `review_queue`, монеты). Эксперт-куратор решает проблему безопасности и качества, которой нет у открытого UGC. Развивает речь ребёнка (не только чтение). Быстро наполняет каталог.

## Пользовательский флоу (детально)

```
ЭКСПЕРТ                         РЕБЁНОК (много детей)              ЭКСПЕРТ
─────────                       ────────────────────              ────────
Создаёт книгу
+ глава 1 (текст/аудио)
   │
   ▼ модерация админа
Публикация в «Сочиняем вместе»
   │
   ▼                           Открывает книгу
                               Читает/слушает главу 1
                               Жмёт «Придумать продолжение»
                               Пишет аудио (до 60 сек)  ──┐
                               (то же делают другие дети) │
                                                          ▼
                                              ИИ: транскрибация + скор
                                                          │
                                                          ▼
                                                   Продолжения падают
                                                   эксперту в очередь
                                                   раунда (round = глава 2)
                                                          │
                                                          ▼
                                              Эксперт слушает, читает скоры,
                                              ВЫБИРАЕТ 1+ лучших, правит текст,
                                              закрывает раунд
                                                          │
              ┌───────────────────────────────────────────┤
              ▼                                             ▼
    ПОБЕДИТЕЛЬ(и):                              ОСТАЛЬНЫЕ:
    + монеты (награда)                          + монеты за участие
    + бейдж «соавтор»                           + мотивирующий текст
    + push «твоё продолжение выбрали!»          + push «новый раунд скоро»
    выбранный текст → глава 2
              │
              ▼
    Раунд глава 3 открывается … цикл …
              │
              ▼
    Эксперт жмёт «Завершить книгу»
    → обложка → публикация в магазин
    → список соавторов
```

## Модель данных

### Стартовая книга эксперта (глава 1) — какие поля заполняются

**Новых полей не требуется** — глава 1 это индекс `[0]` существующих массивов `Challenge`. Стартовая книга и готовая книга имеют одинаковую структуру, конвертация не нужна.

| Что вводит эксперт | Поле в `Challenge` | Примечание |
|---|---|---|
| Название книги | `bookTitle` | ✅ существует |
| Подпись автора («Сочинили вместе») | `bookAuthor` | ✅ существует |
| Обложка (картинка книги) | `coverImage` | ✅ существует — для магазина/списка |
| Название главы 1 | `partTitles[0]` | ✅ существует |
| Текст начала (первые предложения) | `partTexts[0]` | ✅ существует |
| Аудиозапись (озвучка главы 1) | `partAudios[0]` | ✅ существует — плеер `PartAudioPlayer` уже есть |
| Иллюстрация к главе 1 | `partImages[0]` | ✅ существует — картинка внутри ридера |
| Флаг соавторства | `collaborative = true` | 🆕 см. ниже |
| Текущий раунд | `currentRound = 1` | 🆕 |
| Приём продолжений открыт | `collabOpen = true` | 🆕 |
| Награда за выбранную главу | `winnerCoins` (300) | 🆕 |

> Две разные картинки: `coverImage` = обложка всей книги; `partImages[i]` = иллюстрация конкретной главы. Каждая выбранная экспертом глава дописывается как `partTexts[i]`, `partTitles[i]`, опц. `partAudios[i]`, `partImages[i]`.

### Новый enum: `ContributionStatus` (`common/enums/contribution-status.enum.ts`)
- [ ] `PENDING = 'pending'` — на рассмотрении эксперта
- [ ] `SELECTED = 'selected'` — выбрано, стало главой
- [ ] `NOT_SELECTED = 'not_selected'` — в этот раз не выбрали

### Новый enum-барьер на `Challenge` (расширяем существующее)
- [ ] `challenge.entity.ts` — добавить `collaborative: boolean` (default `false`) — это книга-соавторство
- [ ] `challenge.entity.ts` — добавить `currentRound: number` (default `1`) — номер главы, которая сейчас набирается
- [ ] `challenge.entity.ts` — добавить `collabOpen: boolean` (default `false`) — открыт ли приём продолжений прямо сейчас
- [ ] `challenge.entity.ts` — добавить `collabCompleted: boolean` (default `false`) — книга завершена и опубликована в магазин
- [ ] `challenge.entity.ts` — добавить `coAuthors: { type: 'child' | 'parent'; id: string }[]` (json, nullable) — накопленный список соавторов (дети и родители) для отображения и роялти. **Полиморфно**, не `coAuthorChildIds`.
- [ ] `challenge.entity.ts` — добавить `winnerCoins: number` (default `300`) — награда за выбранную главу (настраивает эксперт)
- [ ] Переиспользуем существующие `partTexts[]`, `partTitles[]`, `partAudios[]`, `partImages[]` — выбранная глава просто **дописывается** в эти массивы. Готовая совместная книга = обычная читаемая книга.

> Заметка по `category`: у `collaborative`-книги `category` остаётся `READING` (после завершения её читают вслух как обычно). Флаг `collaborative` — отдельная ось, не трогаем `ChallengeCategory`.

### Новая entity: `ChapterContribution` (`sessions/entities/chapter-contribution.entity.ts`)
- [ ] `id` (uuid)
- [ ] `challengeId` + ManyToOne Challenge (onDelete CASCADE)
- [ ] `roundNumber: number` — для какой главы это продолжение (= `challenge.currentRound` на момент записи)
- [ ] **`authorType: 'child' | 'parent'`** — кто записал продолжение (полиморфно, как `fromType` в монетах). БЕЗ FK, чтобы поддержать обе роли.
- [ ] **`authorId: string`** — id ребёнка или родителя (без жёсткого FK; резолвим в сервисе по `authorType`)
- [ ] `audioUrl: string`
- [ ] `durationSec: number`
- [ ] `transcription: text` (nullable) — результат ИИ
- [ ] `aiScore: decimal(5,2)` (nullable) — общий скор-подсказка 0–10
- [ ] `aiScoreBreakdown: jsonb` (nullable) — `{ relevance, creativity, coherence, language }` (см. §5)
- [ ] `aiFeedback: text` (nullable)
- [ ] `safetyFlag: boolean` (default `false`) — ИИ отметил недетский контент/персданные → скрыть от публичного показа
- [ ] `status: ContributionStatus` (default `PENDING`)
- [ ] `expertEditedText: text` (nullable) — если эксперт отредактировал выбранный текст
- [ ] `coinsAwarded: number` (default `0`)
- [ ] `isWinner: boolean` (default `false`)
- [ ] `createdAt`, `resolvedAt` (nullable)

> Альтернатива, которую сознательно **не** берём: пихать продолжения в `sessions` через новую фазу. Отклонено — продолжение не привязано к `enrollment`/чтению, нет `coinsPerPart`. Отдельная entity чище.

## Backend

### AI (`ai/ai.service.ts`) — новый режим оценки продолжения
- [ ] `assessContribution(previousChapterText, transcription)` — генеративная оценка (см. §5). **НЕ** LCS: у творческого продолжения нет эталона.
- [ ] Переиспользовать существующую транскрибацию аудио (Gemini 2.5 Flash → Whisper-1 резерв).
- [ ] Промпт возвращает JSON: `{ relevance, creativity, coherence, language, overall, safetyFlag, feedback }`.

### Contributions service (`sessions/contributions.service.ts` — новый, или в `challenges`)
- [ ] `createContribution(author: {type, id}, challengeId, audio)` — проверка `collaborative && collabOpen`; загрузка аудио в MinIO (бакет `barsum-audio`); фиксация `durationSec`; статус `PENDING`. Работает и для ребёнка, и для родителя — различие только в `authorType`.
- [ ] Асинхронно (как чтение, через `setImmediate`/очередь): транскрибация → `assessContribution` → запись `aiScore/aiScoreBreakdown/safetyFlag/aiFeedback`.
- [ ] `findRoundContributions(challengeId, roundNumber)` — для эксперта, сортировка по `aiScore` desc; `safetyFlag=true` помечаются отдельно/прячутся.
- [ ] `selectWinners(challengeId, roundNumber, contributionIds[], editedTexts?)` — атомарно:
  - [ ] выбранным: `status=SELECTED`, `isWinner=true`, текст (`expertEditedText` или транскрипция) → **дописать** в `challenge.partTexts[]` (+ `partTitles[]` = «Глава N»);
  - [ ] награда победителю **зависит от роли** (см. §6): `authorType='child'` → монеты `winnerCoins`; `authorType='parent'` → только бейдж «соавтор», без монет;
  - [ ] добавить `{type: authorType, id: authorId}` в `challenge.coAuthors` (без дублей);
  - [ ] остальным продолжениям раунда: `status=NOT_SELECTED`; детям — `CONTRIBUTION_PARTICIPATION_COINS`; родителям — без монет (только мотивирующий текст);
  - [ ] `challenge.currentRound += 1`; при желании закрыть/оставить `collabOpen`.
- [ ] `completeBook(challengeId, coverImage?)` — `collabCompleted=true`, `collabOpen=false`; если задан/сгенерирован `coverImage` — записать; книга становится доступной в магазине.
- [ ] `openRound(challengeId)` / `closeRound(challengeId)` — `collabOpen` on/off (ручное управление приёмом).

### Coins (`coins/coins.service.ts`)
- [ ] Монеты начисляются **только детям** (`toType='child'`). Существующий `transfer()` с типом `EARN`, новых типов не нужно. Различаем по `referenceType`: `'collab_win'` | `'collab_participation'`. Идемпотентность по `referenceId=contributionId`.
- [ ] Родитель монеты **не получает** (он эмитент экономики) — его награда за победу это статус «соавтор». См. §6.
- [ ] Константы: `CONTRIBUTION_PARTICIPATION_COINS = 50` (участие ребёнка), `winnerCoins` — из `challenge` (default 300, задаёт эксперт).

### Controller (`sessions/contributions.controller.ts` или в challenges)
- [ ] `POST /challenges/:id/contributions` (child **или** parent, multipart audio) — записать продолжение; `authorType/authorId` берутся из JWT (не из тела)
- [ ] `GET /challenges/:id/contributions?round=N` (expert) — очередь раунда со скорами
- [ ] `GET /challenges/:id/contributions/mine` (child **или** parent) — мои продолжения и их статусы (для мотивирующего экрана); фильтр по `authorType/authorId` из JWT
- [ ] `POST /challenges/:id/rounds/:round/select` (expert) — выбрать победителей `{ contributionIds, editedTexts? }`
- [ ] `POST /challenges/:id/complete` (expert) — завершить книгу `{ coverImage? }`
- [ ] `POST /challenges/:id/round/open` · `POST /challenges/:id/round/close` (expert)
- [ ] `GET /challenges?collaborative=true&open=true` — список книг, открытых для соавторства (child)

### Уведомления (переиспользуем Web Push, VAPID — уже есть)
- [ ] Победителю: push «🎉 Твоё продолжение выбрали! Ты соавтор книги "…"»
- [ ] Участникам раунда (не выбранным): push «Новая глава готова — придумай продолжение!» (мягкий возврат)
- [ ] Всем участникам книги при открытии нового раунда: push «Открылась глава N — сочиняем дальше!»

## §5. ИИ-скоринг продолжения (метрики)

**Важно:** это НЕ оценка чтения. При чтении есть эталонный текст → LCS считает точность. У продолжения **эталона нет** → скор строится генеративной моделью и служит **подсказкой эксперту**, финальное решение всегда за человеком.

`assessContribution(previousChapterText, transcription) → { … }`:

| Метрика | 0–10 | Что оценивает |
|---|---|---|
| `relevance` | связь продолжения с предыдущей главой (не «улетел» в сторону) |
| `creativity` | оригинальность, интересность идеи |
| `coherence` | связность, законченность мысли, логика |
| `language` | богатство речи, грамотность, словарный запас |
| `overall` | взвешенная итоговая (напр. relevance·0.3 + creativity·0.3 + coherence·0.2 + language·0.2) |
| `safetyFlag` | bool: недетский контент / имена-адреса / персданные → скрыть |
| `feedback` | 1–2 фразы для эксперта: сильные/слабые стороны |

- [ ] Порога авто-выбора **нет** — выбирает эксперт. Скор только сортирует и подсвечивает.
- [ ] `safetyFlag=true` → продолжение прячется из общего списка, помечается для эксперта отдельно.

## §6. Монеты и награды (с учётом ролей)

| Роль автора | Победа (SELECTED) | Участие (NOT_SELECTED) |
|---|---|---|
| **Ребёнок** | `winnerCoins` (default 300) + бейдж «соавтор» | `CONTRIBUTION_PARTICIPATION_COINS = 50` + мотив. текст |
| **Родитель** | бейдж «соавтор» (без монет) | мотив. текст (без монет) |

- [ ] Монеты — **только детям**: `EARN`, `referenceType='collab_win'` / `'collab_participation'`.
- [ ] Родитель — эмитент экономики, монеты ему не начисляются; его награда за победу это **статус соавтора** (можно позже: бонусные монеты на счёт его ребёнка или скидка — вне этой задачи).
- [ ] Ребёнок никогда не в минусе — участие всегда даёт монеты. Ключ к возврату.
- [ ] Идемпотентность по `referenceId = contributionId` (защита от двойного начисления при повторе выбора).
